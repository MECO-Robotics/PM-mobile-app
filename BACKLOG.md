# PM Mobile App Backlog

This backlog tracks practical, function-first improvements for the robotics team app that do not require sign-in, roles, or permissions.

## Completed

- Home next-action list for urgent tasks, purchases, and manufacturing.
- Required-field validation across core editors.
- Safer blocker clearing with a resolution note.
- QA reports create follow-up work for non-pass outcomes.
- Event reports can create follow-up tasks.
- Task dependencies, dependency picker, search, circular dependency prevention, and readiness warnings.
- Task cards show logged hours, estimate overruns, dependency status, and quick work logging.
- Task filters for blockers, due soon, overdue, ready now, dependency wait, fabrication, purchase, and unassigned work.
- Automatic task status movement when work starts, blockers clear, dependencies complete, or QA passes.
- Request QA action from eligible task cards.
- Need-help requests from active tasks and work logs appear in the mentor report queue without changing task or work-log state.
- QA request list can open a report and clears completed requests.
- QA reports preserve exact request metadata when opened from a request.
- Manually created QA requests can link to existing tasks.
- Ready-for-QA task filter and summary count.
- QA reports include evidence/reference notes.
- Work-log note templates for CAD, machining, wiring, programming, testing, and meetings.
- Task cards can copy an existing task into a new draft for repeated work.
- Visible open tasks can have due dates shifted in bulk from the task queue.
- Tasks support simple checklist/substep notes and show them on queue cards.
- Subsystem cards show health from blockers, overdue work, QA findings, waiting QA, and open purchases.
- Materials show open purchase coverage and suggested restock quantities.
- Home includes a CSV-style meeting export for priority tasks, purchases, and manufacturing.
- Task queue empty state explains stacked filters and offers reset/add actions.
- Reduced lint warnings by removing unused `App.tsx` imports and dead constants.
